SECTION 01

What Is This Project?
Right now, most HR teams track their performance numbers using Excel files. These get emailed around, go out
of date quickly, and require someone to manually chase people for updates. Mistakes happen. Things get
missed.
The HR KPI Board fixes all of this. It is a simple web app where:
• Employees log in and see only their own tasks and numbers.
• Managers see their whole team's performance on one screen.
• Admins control everything — set targets, manage users, run reports.
• Everything updates automatically the moment someone submits a new number.
• The system flags problems without anyone having to manually check.
Think of it as a live scoreboard for HR — always up to date, always accessible from any phone or computer.

SECTION 03

Who Uses It and What They Can Do
■ Employee

Role

• See their own KPI targets and current scores.
• Submit new KPI values using a simple form on their phone.
• Update task status: Pending → In Progress → Done.
• Receive reminders when deadlines are approaching.

■ Manager

Role

• See everything an Employee sees, plus the full team's performance.
• View a leaderboard showing who is on track and who is not.
• See trend indicators — is each person improving, declining, or stable?
• Get automatic alerts when a team member's KPI goes Off Track.

■■ Admin

Role

• Full access to every part of the system.
• Set KPI targets, weights, and categories for the organisation.
• Add or remove users and assign roles.
• Generate and export reports to PDF or Excel in one click.
• Configure notifications, escalation rules, and automation settings.

SECTION 05

The 4 Build Phases — Step by Step
Phase 0 — Foundation & Setup

Weeks 1–2

Before building anything users see, we lay a solid technical foundation. Like laying bricks before building the
house — get this right and everything else is faster.
• Set up the Supabase database with 5 core tables: Users, KPIs, KPI Submissions, Tasks, and
Notifications.
• Configure Row-Level Security (RLS) so employees can ONLY see their own data — not other people's.
• Scaffold the React project and connect it to the database.
• Deploy to Vercel so the team has a live staging link from day one.
• Set up Git version control so multiple developers can work without overwriting each other.
DELIVERABLE

A working (but empty) app accessible on a live staging URL.

Phase 1 — Full MVP

Weeks 3–8

The main build. Everything employees, managers, and admins need for their day-to-day work gets built in this
phase.
• Login screen with automatic role detection — the right dashboard appears for each user.
• KPI Dashboard with 7 KPI cards, traffic-light status, and an overall health score.
• Real-time updates — when someone submits a number, everyone's dashboard refreshes instantly.
• Task Management — employees update task status from their phone; overdue tasks are flagged
automatically.
• KPI Submission Form — a clean form for employees to enter values with optional notes.
• Team Leaderboard — managers see all team members ranked with improving/declining arrows.
• Push notifications via OneSignal — deadline reminders and Off Track alerts.
DELIVERABLE

A fully working app the whole team uses in their daily work.

Phase 2 — Automation Layer

CONFIDENTIAL

Weeks 9–12

Now we make the app work for the team instead of the other way around. The system starts doing things
automatically so people can focus on decisions, not data entry.
• Auto-score calculation — Edge Functions recalculate the health score on every new submission.
• AI narrative summaries — a one-line auto-generated insight per KPI, e.g. 'Turnover rose 2% this month
— flagged for review'.
• Monthly target generation — the system suggests next month's targets based on historical trends.
• One-tap export — admins download a full quarterly report as PDF or Excel in a single click.
• Escalation alerts — if any KPI stays Off Track for more than 7 days, a senior manager is automatically
notified.
DELIVERABLE

A largely self-running system that surfaces problems before anyone has to manually check.

Phase 3 — Native App & Enterprise

Weeks
13–14+

The final phase takes the product to enterprise scale and puts it in the app stores.
• Wrap the app using Capacitor to publish on the Apple App Store and Google Play Store.
• Add SSO (Single Sign-On) so staff log in with their existing company email accounts.
• Connect to HRIS platforms (SAP, Workday, BambooHR) to auto-import employee data.
• Advanced analytics: trend charts, year-over-year comparisons, predictive modelling.
• Custom branding: your logo, colour scheme, and a dedicated domain name.
DELIVERABLE

A fully branded, enterprise-ready HR platform available on all devices and in app stores.

SECTION 07

Key Risks — Get These Right Early
Security Rules (RLS)

■

Database row-level security must be set correctly in Weeks 1–2. If wrong, employees could see other
people's data. This is the most critical technical task in the whole project.

Role Design

■

The three roles (Employee, Manager, Admin) must be defined clearly before any screen is built.
Changing role logic midway through development is expensive and time-consuming.

Real-Time Subscriptions

■

Live dashboard updates must be scoped so employees only subscribe to their own data — not the
whole company's dataset — or the app will be slow for everyone.

AI Summary Costs

■

The AI narrative feature calls an external API that charges per request. Trigger it once per KPI
submission only — never on every page load — to keep monthly costs predictable.
